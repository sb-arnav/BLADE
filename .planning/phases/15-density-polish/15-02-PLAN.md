Ship a clear four-tier visual hierarchy across the top bar (TitleBar + ActivityStrip) that fits at 1280px width with no overflow and no competing visual weight.

Purpose: Tester grievance #5 — "top bar feels cluttered, no hierarchy". /* Hierarchy tier semantics:
 * tier=1  primary (BLADE brand, traffic lights)         — color: var(--t-1), weight 600+
 * tier=2  secondary (status pill, activity summary)     — color: var(--t-2), weight 500
 * tier=3  tertiary (⌘K hint)                            — color: var(--t-3), weight 400
 * tier=4  background chrome (dividers, fills)           — color: var(--t-4), no text
 *
 * Controls visual weight so future additions cannot silently compete.
 */
.titlebar [data-hierarchy-tier="1"] .titlebar-brand {
  color: var(--t-1);
  /* existing weight 600 preserved */
}
.titlebar [data-hierarchy-tier="2"],
.titlebar-status[data-hierarchy-tier="2"] {
  color: var(--t-2);
  font-weight: 500;
}
.titlebar [data-hierarchy-tier="3"],
.titlebar-hint[data-hierarchy-tier="3"] {
  color: var(--t-3);
  font-weight: 400;
  opacity: 0.78;
}

/* 1280px width guardrail — tighten gap and hide optional-tier if overflow looms.
 * Ensures no horizontal overflow under 1280px with both status + activity + ⌘K. */
@media (max-width: 1280px) {
  .titlebar {
    padding: 0 var(--s-2);

  .titlebar-title {
    gap: var(--s-2);
  }
}

/* Below 1100px, hide ⌘K hint entirely (tier 3 is disposable). */
@media (max-width: 1100px) {
  .titlebar-hint[data-hierarchy-tier="3"] {
    display: none;
  }
}
