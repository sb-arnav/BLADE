**Step A — Add a `score_or_default` helper in brain.rs that wraps `score_context_relevance` in `catch_unwind`. Place near `score_context_relevance` (around L218-L320):**

    /// Phase 32 / CTX-07 — panic-resistant wrapper around score_context_relevance.
    /// If scoring panics (e.g. NaN-related arithmetic, future code changes that
    /// regress the function, or test-injected panics), returns the `safe_default`
    /// value. Default 1.0 means "inject this section" — i.e. degrade to the
    /// pre-Phase-32 naive path. This is the v1.1 lesson incarnate: the smart
    /// path must NEVER crash the dumb path.
    pub fn score_or_default(query: &str, context_type: &str, safe_default: f32) -> f32 {
        let result = std::panic::catch_unwind(|| {
            score_context_relevance(query, context_type)
        });
        match result {
            Ok(score) if score.is_finite() => score,
            Ok(_) => {
                // NaN or infinity from scoring is a bug; record and degrade to safe.
                crate::trace::log_trace(&crate::trace::TraceEntry::ctx07_fallback(
                    "score_context_relevance returned non-finite",
                ));
                safe_default
            }
            Err(_) => {
                // Panic was caught by catch_unwind. Log and degrade.
                crate::trace::log_trace(&crate::trace::TraceEntry::ctx07_fallback(
                    "score_context_relevance panicked",
                ));
                safe_default
            }
        }
    }
