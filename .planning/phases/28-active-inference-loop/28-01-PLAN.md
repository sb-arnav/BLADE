**Normalization (per D-04):**
`pub fn normalize_error(expected: f32, observed: f32, range_max: f32) -> f32` — returns `((expected - observed).abs() / range_max.max(1.0)).min(1.0)`.

`pub fn prediction_error_for_tentacle(pred: &TentaclePrediction, observed: &HashMap<String, f32>) -> f32` — weighted mean of per-signal `normalize_error()` weighted by `confidence`. Returns 0.0 if no matching signals.

**EMA learning (per D-15, D-16):**
`pub(crate) fn update_prediction_ema(prediction: &mut TentaclePrediction, observed: &HashMap<String, f32>)` — per-tentacle alpha: `"calendar"` -> 0.1, `"slack"` -> 0.08, `"github"` -> 0.05, default -> 0.07. For each signal in observed that exists in prediction.signals: `entry.expected = entry.expected * (1.0 - alpha) + obs_val * alpha`. Also increase confidence: `entry.confidence = (entry.confidence + 0.02).min(1.0)`.

**Aggregate computation (per D-08 — Error/Dormant = 0 weight):**
`fn compute_aggregate_error(errors: &HashMap<String, f32>) -> (f32, String)` — queries `crate::hive::get_hive_status().tentacles` to build a `HashMap<String, TentacleStatus>` keyed by platform name (strip `"tentacle-"` prefix from each `TentacleSummary.id`). For each entry in `errors`:
- Look up the tentacle's status. If status is `TentacleStatus::Error` or `TentacleStatus::Dormant` or `TentacleStatus::Disconnected`, assign weight = 0.0.
- If status is `TentacleStatus::Active` (or if the platform is `"calendar"` which has no Hive tentacle and is always treated as Active), assign weight = 1.0.
- Compute weighted mean: `sum(error * weight) / sum(weight)`. If total weight is 0.0, return (0.0, "none").
- Return (aggregate_error, platform_with_highest_error). The "top tentacle" is determined from all entries (including 0-weight ones), so the user can see which tentacle has the highest error even if it's dormant.

This implements D-08 fully: "Aggregate prediction error = weighted mean across active tentacles; Error/Dormant tentacles contribute 0 weight."

**Test-only aggregate helper:**

**Main entry point (per D-09, called by hive_tick in Plan 02):**
`pub async fn compute_prediction_errors(app: &tauri::AppHandle, reports: &[crate::hive::TentacleReport])` —
1. Extract signals from each report via `extract_signals_from_report()`. Group by platform (use tentacle_id prefix: `"tentacle-slack"` -> `"slack"`, `"tentacle-github"` -> `"github"`).
2. Also extract calendar signals via `extract_calendar_signals(&crate::integration_bridge::get_integration_state().upcoming_events)` (always, since calendar is not a Hive tentacle).
3. Lock PREDICTIONS. For each platform with observed signals: get or insert default prediction, compute error via `prediction_error_for_tentacle()`, update EMA via `update_prediction_ema()`, collect (platform, error) pairs.
4. Compute aggregate error + top tentacle via `compute_aggregate_error()` (which applies D-08 weight filtering internally).
5. Detect single-tentacle spike: any tentacle error > 0.7 while others < 0.3.
6. Update SUSTAINED_HIGH_TICKS: if aggregate > 0.6 -> fetch_add(1), else -> store(0).
7. Call `crate::homeostasis::update_physiology_from_prediction_errors(aggregate_error, sustained_ticks, is_single_spike)`.
8. Persist predictions to SQLite via `persist_predictions_to_db()`.
9. Log error to prediction_error_log via `log_prediction_error()` (prune BEFORE insert per Pitfall 5).
10. If aggregate > 0.5, emit ActivityStrip event with module="active_inference", action="prediction_error_threshold".
