**Band computation (per D-04, D-11):**
- `fn initial_band_from_scalar(scalar: f32) -> VitalityBand` -- simple threshold match for cold DB load (no hysteresis): >=0.6 Thriving, >=0.4 Waning, >=0.2 Declining, >=0.1 Critical, else Dormant
- `fn compute_band(scalar: f32, current_band: &VitalityBand) -> VitalityBand` -- hysteretic transitions per D-11. Moving DOWN crosses threshold. Moving UP requires threshold + HYSTERESIS_BUFFER (0.05). Match on current_band:
  - Thriving: if scalar < 0.6 -> Waning, else stay
  - Waning: if scalar >= 0.65 -> Thriving, if scalar < 0.4 -> Declining, else stay
  - Declining: if scalar >= 0.45 -> Waning, if scalar < 0.2 -> Critical, else stay
  - Critical: if scalar >= 0.25 -> Declining, if scalar <= 0.0 -> Dormant, else stay
  - Dormant: stays Dormant (only changed by reincarnation)

**SDT replenishment computation (per D-12, D-13):**
- `fn compute_replenishment() -> SDTSignals`:
  - **Competence:** `crate::reward::read_reward_history(10)`, compute simple average of `.reward` fields. If empty, default 0.5. Divide by 0.7 and clamp to [0.0, 1.0]. Score > 0.7 = full competence signal.
  - **Autonomy:** `crate::decision_gate::get_decision_log()`, take last 20. Count entries where outcome matches `DecisionOutcome::ActAutonomously { .. }` AND feedback != Some(false). Ratio = autonomy signal. If empty, default 0.5.
  - **Relatedness:** For now, use session activity tracking: count messages from the in-memory conversation buffer (if available) or default 0.5. Positive feedback from character.rs can be read from the `reactions` SQLite table directly: `SELECT COUNT(*) FROM reactions WHERE polarity > 0 AND timestamp > ?` (trailing 1-hour window). Message frequency: count messages from `chat_history` table in the last hour, cap at 10. Average message length check: `SELECT AVG(LENGTH(content)) FROM chat_history WHERE role='user' AND timestamp > ?`. Composite: `(msg_freq_capped/10.0 * 0.4) + (positive_reactions_normalized * 0.3) + (avg_length_signal * 0.3)` where avg_length_signal = min(1.0, avg_len / 50.0).
  - **Net:** `0.4 * competence + 0.3 * autonomy + 0.3 * relatedness`

**Drain computation (per D-14, D-15, D-16):**
- `fn compute_drain() -> DrainSignals`:
  - **Failure drain:** Read latest reward via `crate::reward::read_reward_history(1)`. If composite < 0.3, drain = (0.3 - score). Else 0.
  - **Eval drain:** Accumulated via `apply_drain()` calls from safety_bundle.rs. Store accumulated eval drain in VitalityState and reset each tick after reading. Track via `pending_eval_drain: f32` field (add to struct).
  - **Isolation drain:** Compare current timestamp to last user message timestamp. If delta > 7200 seconds (2 hours), drain = 0.01. Read last message time from `chat_history` table: `SELECT MAX(timestamp) FROM chat_history WHERE role='user'`.
  - **Prediction error drain:** `crate::active_inference::get_active_inference_state().aggregate_error`. If > 0.6, increment `sustained_high_error_ticks` in VitalityState. If sustained_high_error_ticks > 5, drain = 0.01 * (sustained_high_error_ticks - 5).min(5) as f32. Reset counter when error drops below 0.6.
  - **Tedium drain:** Read last 5 user messages from chat_history. If fewer than 2, skip. Use `crate::embeddings::embed_texts()` to embed them. Compute pairwise cosine similarity (inline the formula: dot product / (norm_a * norm_b) since cosine_similarity is private in embeddings.rs). If average similarity > 0.85, drain = 0.005. Cache the embedding result in a `last_tedium_embeddings: Option<Vec<Vec<f32>>>` field to avoid re-embedding every tick. Only re-compute if message buffer changed (check latest message timestamp against a stored `last_tedium_check_ts`).
  - **Net:** Sum of all channels.

Calibrate drain rates so that with ALL channels maximally active (failure 0.3 + eval 0.02 + isolation 0.01 + prediction 0.05 + tedium 0.005 = ~0.385 per tick), it takes roughly 2+ hours of 60s ticks to drain from 1.0 to 0.0 (~120 ticks). Scale the replenishment and drain rates accordingly. A reasonable approach: multiply net_drain by a scaling factor of 0.025 so maximum drain per tick is ~0.01.

**Drain floor (per D-16):**
- `static DRAIN_FLOOR_TICKS: AtomicU32` -- counts consecutive ticks at scalar <= 0.05 with zero replenishment
- Vitality cannot drop below 0.05 from drain alone. Final step to 0.0 requires EITHER 3 consecutive ticks at 0.05 with zero replenishment net, OR explicit user command (exposed as `pub fn force_dormancy()` tauri command).

**Vitality tick (per D-03):**
- `pub fn vitality_tick()` -- called from hypothalamus_tick() (Plan 02 wires this):
  1. Acquire VITALITY lock
  2. Compute SDT replenishment via `compute_replenishment()`
  3. Compute drain via `compute_drain()`
  4. net_delta = (net_replenishment_scaled) - (net_drain_scaled), scaled so max drain per tick < 0.01
  5. new_scalar = (scalar + net_delta).clamp(0.0, 1.0) -- f32 NaN guard: if new_scalar.is_nan() { 0.5 }
  6. Apply drain floor: if new_scalar < 0.05 and drain is causing the drop, clamp to 0.05. Increment DRAIN_FLOOR_TICKS. If DRAIN_FLOOR_TICKS >= 3 and net replenishment is zero, allow drop to 0.0.
  7. Compute new band via `compute_band(new_scalar, &current_band)` -- hysteretic
  8. Detect band change: `old_band != new_band`
  9. Update trend: `trend = new_scalar - old_scalar`
  10. Push VitalitySnapshot to history ring buffer (pop_front if len >= HISTORY_RING_MAX)
  11. Update all fields in VitalityState
  12. Release lock
  13. Persist to SQLite: `persist_vitality(&state)` and `persist_vitality_history(&state)`
  14. If band changed: emit ActivityStrip event via `emit_band_transition(old_band, new_band, new_scalar)`
  15. Emit blade_vitality_update if band changed OR abs(trend) > 0.05 since last emission
  16. If new_band == Dormant: trigger dormancy sequence

**Dormancy sequence (per D-17):**
- `fn trigger_dormancy()`:
  1. Serialize full state to SQLite (vitality_state table updated)
  2. Get app handle from VITALITY_APP
  3. Emit `blade_dormancy` event: `{ reincarnation_count, top_drain_factors: [...], total_uptime_secs, vitality_at_dormancy }`
  4. Write dormancy_record to SQLite: timestamp, descent_history (last 20 snapshots as JSON), top_drain_factors, session_count=0, reincarnation_completed=0
  5. If DORMANCY_STUB.load(Ordering::SeqCst): `log::warn!("[vitality] DORMANCY_STUB active -- skipping process exit");` return
  6. Else: `std::thread::sleep(Duration::from_secs(5));` then `std::process::exit(0);`

**Reincarnation (per D-18):**
- `pub fn check_reincarnation(app: AppHandle)`:
  1. Query SQLite: `SELECT id, reincarnation_count FROM dormancy_records WHERE reincarnation_completed = 0 LIMIT 1`
  2. If no row found, return (normal start)
  3. Load all preserved memory, persona, skills (they persist in their own tables -- no action needed, they're already there)
  4. Reset hormones: call `crate::homeostasis::reset_to_defaults()` (this function may need to be added -- if it doesn't exist, reset by writing default JSON to settings key)
  5. Set vitality scalar = 0.3, band = Declining (via initial_band_from_scalar)
  6. Increment reincarnation_count
  7. Set needs_reincarnation_context = true (brain.rs reads this in Plan 02)
  8. Persist updated state
  9. Mark dormancy record: `UPDATE dormancy_records SET reincarnation_completed = 1 WHERE id = ?`
  10. Emit `blade_reincarnation` event: `{ reincarnation_count, vitality_start: 0.3, memories_intact: true }`
