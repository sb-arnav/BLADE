In src/design-system/primitives/Dialog.tsx — fix focus management for A11Y2-04:
  - The native <dialog>.showModal() traps focus within the dialog (browser behavior, confirmed).
  - Problem: on open, focus does not automatically move to the first interactive child — it stays on the trigger element outside the dialog when opened via showModal() in some WebView2 versions.
  - Problem: on close, focus does not return to the trigger element that opened the dialog.
  - Fix:
    a. Accept an optional triggerRef?: React.RefObject<HTMLElement> prop
    b. Capture document.activeElement BEFORE calling el.showModal() and store it in a ref (prevFocusRef)
    c. After showModal(), call the first focusable child's .focus() — find it with el.querySelector('button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])')
    d. On close (when open flips false), call prevFocusRef.current?.focus()
  - Add aria-modal="true" to the <dialog> element
  - Keep existing aria-label prop wiring
  - Keep existing onClose handler
  - TypeScript: update DialogProps to add triggerRef?: React.RefObject<HTMLElement>
