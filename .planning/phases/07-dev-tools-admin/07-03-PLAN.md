<threat_model>
## Trust Boundaries

- **User input → run_shell boundary**: Terminal input flows directly into `native_tools::run_shell`. Rust-side is responsible for sandboxing/escaping; frontend does NO shell-escaping (would be double-escape risk).
- **User input → file path boundary**: FileBrowser preview path from tree click flows into `fileRead`. Rust-side validates; frontend no-escape.
- **User input → browser action boundary**: Canvas code flows into `sandboxRun`; sandbox is sandboxed in Rust.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Elevation of Privilege | Terminal `runShell` executes arbitrary shell commands | accept | Single-user desktop app (local-first); user typing IS authorization. Backend logs all shell invocations via execution_memory. |
| T-07-03-02 | Information Disclosure | FileBrowser preview exposes file contents | accept | Same local-first posture; user clicking IS authorization. |
| T-07-03-03 | Tampering | Git style clear action without confirmation | mitigate | `gitStyleClear` behind Dialog confirm (D-174). |
| T-07-03-04 | Denial of Service | WorkflowBuilder runs long-running workflow | accept | `workflowRunNow` returns immediately; runs poll. User can cancel via pause/cancel if exposed (not in Rust surface — add if needed in Phase 9). |
| T-07-03-05 | Tampering | WorkflowDelete without confirm | mitigate | Dialog-confirm with `variant="danger"` button per Pattern §4. |
</threat_model>
