wire-up sites + 6 tests + 1 reward_weights test fix). Plan 34-01 was running
in parallel and had `git add`'d its config.rs work to the index before

and `commands.rs`, but `git commit` (without `--only`) commits everything
staged in the index â€” so the pre-staged config.rs rode along.
**Resolution applied:** Left as-is. The committed content IS Plan 34-01's
correct work (verified: `git diff HEAD -- src-tauri/src/config.rs` is empty),
and rewriting history would risk destroying Plan 34-01's parallel progress
or introducing rebase conflicts. The attribution is slightly muddled (Plan
34-01's content appears under Plan 34-02's commit message) but the substantive
state of the tree is correct.
**Future mitigation:** Use `git commit --only <paths>` instead of `git add
<paths> && git commit` when running parallel-plan executors that share index
state, OR use a worktree per parallel plan. Plan 34-01 then committed its
SUMMARY at `04fa671` (between my Task 2 and Task 3 commits) so the SUMMARY
attribution is now consistent end-to-end.
**Files affected:** `src-tauri/src/config.rs` (no Plan 34-02 content; pure
Plan 34-01 work)
