<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React render → ErrorBoundary catch | untrusted component errors (any child throw) bubble up to ErrorBoundary render path |
| ErrorBoundary → clipboard | error.message + componentStack written to clipboard on "Copy error" click |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-02-01 | Information disclosure | ErrorBoundary "Copy error" | accept | error.message + componentStack are LOCAL to the user; clipboard ops in Tauri desktop context have no remote exfiltration vector. If developer builds include source paths, those leak to clipboard when user copies — acceptable since clipboard is user-initiated. |
| T-09-02-02 | Denial of service | Infinite error loop in ErrorBoundary render | mitigate | Error state is local to the boundary; hitting "Reset route" sets state back to { error: null }. If the same render keeps throwing, resetKey (route.id) change on navigation auto-clears. No infinite-render loop possible. |
| T-09-02-03 | Elevation of privilege | ErrorBoundary "Back to dashboard" hash navigation | accept | Hash navigation changes URL, does NOT bypass onboarding gate. The onboarding gate in MainShell pre-empts RouteSlot render. |
| T-09-02-04 | Tampering | EmptyState actionLabel/onAction injection | accept | Consumer-controlled; consumers pass trusted callbacks. No dynamic code paths in EmptyState itself. |
</threat_model>
