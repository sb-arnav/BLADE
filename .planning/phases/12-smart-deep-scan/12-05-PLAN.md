    ```javascript
    #!/usr/bin/env node
    // verify-scan-event-compat.mjs — D-64 onboarding compat invariant:
    // Every phase name in DEEP_SCAN_PHASES must have a Rust emit site in the new scanner.
    // Prevents silent regressions where a renamed phase breaks the onboarding progress ring.
    import { readFileSync, readdirSync } from 'fs';
    import { join, extname } from 'path';
    import { exit } from 'process';

    // Read the TypeScript phase list
    const phasesFile = new URL('../src/features/onboarding/deepScanPhases.ts', import.meta.url).pathname;
    const phasesSource = readFileSync(phasesFile, 'utf8');

    // Extract DEEP_SCAN_PHASES array entries using a simple regex
    // Matches: 'starting', 'installed_apps', etc. inside the const array
    const matches = phasesSource.match(/'([a-z_]+)'/g) || [];
    const phaseNames = matches.map(m => m.replace(/'/g, ''));

    if (phaseNames.length === 0) {
      console.error('[FAIL] verify:scan-event-compat: could not extract phase names from deepScanPhases.ts');
      exit(1);
    }

    // Read all Rust source files under src-tauri/src/deep_scan/
    const scanDir = new URL('../src-tauri/src/deep_scan', import.meta.url).pathname;

    function collectRsFiles(dir) {
      let files = [];
      try {
        for (const entry of readdirSync(dir, { withFileTypes: true })) {
          const full = join(dir, entry.name);
          if (entry.isDirectory()) files = files.concat(collectRsFiles(full));
          else if (entry.isFile() && extname(entry.name) === '.rs') files.push(full);
        }
      } catch (_) {}
      return files;
    }

    const allRustSource = collectRsFiles(scanDir)
      .map(f => readFileSync(f, 'utf8'))
      .join('\n');

    let failed = false;
    for (const phase of phaseNames) {
      // Check that this phase name appears as a string literal in the Rust emit sites.
      // The emit site looks like: emit_progress(&app, "phase_name", ...) or "phase": "phase_name"
      if (!allRustSource.includes(`"${phase}"`)) {
        console.error(`[FAIL] verify:scan-event-compat: phase "${phase}" from DEEP_SCAN_PHASES not found in any Rust emit site in deep_scan/`);
        failed = true;
      }
    }

    if (!failed) {
      console.log(`[PASS] verify:scan-event-compat: all ${phaseNames.length} phase names have Rust emit sites`);
    }
    exit(failed ? 1 : 0);
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| enrichment.rs → providers gateway | Scan data (account handles, path names) is sent as LLM prompt content; content must not include secrets from shell history |
| LLM response → llm_narrative.json | LLM-generated text stored on disk; no execution of the content; display-only |
| verify-scan-event-compat.mjs | Reads TS + Rust source files; no user input; no shell injection surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-22 | Information Disclosure | LLM prompt containing account handles | accept | Account handles (github/arnav) are already stored in scan_results.json; sending them to the configured LLM provider is within the expected trust boundary (user chose the provider). D-61 explicitly sanctions this. |
| T-12-23 | Denial of Service | LLM call hangs indefinitely | mitigate | Route through providers/mod.rs gateway which has existing timeout handling (confirmed by reading providers/mod.rs). If no timeout: enrichment.rs uses `tokio::time::timeout(Duration::from_secs(30), llm_call)` around each call. |
| T-12-24 | Information Disclosure | llm_narrative.json readable | accept | Contains only LLM-generated narrative summaries (not the raw account/path data). Lower sensitivity than scan_results.json. Same file permissions (umask from directory). |
| T-12-25 | Spoofing | verify-scan-event-compat false-pass | mitigate | Gate checks string literal presence in Rust source — exact string `"phase_name"` must appear. A renamed phase without updating deepScanPhases.ts → gate fails. A renamed phase WITH updating deepScanPhases.ts → no regression (both sides match). |
| T-12-26 | Information Disclosure | enrichment prompt including shell history content | mitigate | Enrichment module builds prompts ONLY from AccountRow + RhythmSignal data — both of which are already sanitized by their respective scanners (shell_history.rs never stores raw lines; git_remotes.rs strips credentials). enrichment.rs must not directly read scan_results.json raw data — it receives the typed DeepScanResults struct. |
</threat_model>
