<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| paste textarea → `parse_provider_paste` command | User-controlled string (may contain secrets, non-ASCII, binary) enters Rust parser |
| parser Err string → UI | Parser error messages echo first 40 chars of input — must not panic on non-ASCII |
| parsed `api_key` → caller | Returned in plaintext inside Result; caller (Plan 11-03) is responsible for storing via keyring, NEVER logging |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | parser regex input | mitigate | All regexes use bounded character classes `[\w\-\.]` for keys; non-matching input falls through to Err without partial state |
| T-11-02 | Information Disclosure | parser error messages | mitigate | Error strings use `crate::safe_slice(input, 40)` — bounded prefix only; never echoes full paste which may contain keys |
