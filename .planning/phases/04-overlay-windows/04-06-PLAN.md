          show({
            type: 'info',
            title: 'Quick ask bridged',
            message: query.length > 80 ? query.slice(0, 80) + '...' : query,
          });
        },
      );
      return null;

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BLADE_QUICKASK_BRIDGED payload (cross-window) | Rust emits to main; main injects user turn |
| BLADE_ROUTE_REQUEST payload (cross-window) | HUD / other windows emit; main routes |
| SHORTCUT_REGISTRATION_FAILED payload | Rust emits to main; toast bridge renders |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | Tampering | BLADE_ROUTE_REQUEST with arbitrary route_id | mitigate | openRoute looks up ROUTE_MAP; unknown ids fall through to DEFAULT_ROUTE_ID fallback in RouteSlot. No security-sensitive side effects. |
| T-04-06-02 | Information Disclosure | QuickAsk query rendered in toast message | mitigate | Query truncated to 80 chars in toast; full query goes into chat messages (already user-owned context). |
| T-04-06-03 | DoS | Rapid BLADE_QUICKASK_BRIDGED events spam messages[] | mitigate | One event per quickask submit; Rust serializes submissions. Frontend dedup is not needed. |
| T-04-06-04 | Tampering | injectUserMessage called with crafted id to collide with streaming | mitigate | Ids come from Rust uuid (Plan 04-01); no user-controlled path into injectUserMessage outside this bridge. |
| T-04-06-05 | Repudiation | User submits quickask but doesn't notice the bridged toast | accept | Toast has 4s duration (Phase 2 default); chat messages persist regardless; user can always open /chat to see. |
</threat_model>
