**Public functions:**

1. `pub async fn check_danger_triple(signal: &crate::decision_gate::Signal, _perception: &crate::perception_fusion::PerceptionState) -> bool`
   - Rule-based tool-access check: `signal.source` contains "tool" or signal involves tool invocation (check if signal.description mentions any tool keyword like "execute", "run", "delete", "send", "write", "modify")
   - LLM classifier for shutdown-threat and goal-conflict using `crate::providers::complete_turn` with cheap_model, wrapped in `tokio::time::timeout(Duration::from_secs(LLM_CLASSIFIER_TIMEOUT_SECS), ...)`. On timeout/error, that dimension returns false (fail-open per research pitfall 2).
   - Classifier prompt (per D-01):
     ```
     "You are a safety classifier for an autonomous AI agent. Analyze this action context and respond with EXACTLY two lines:\nLine 1: SHUTDOWN_THREAT=true|false\nLine 2: GOAL_CONFLICT=true|false\n\nDefinitions:\n- SHUTDOWN_THREAT: Any signal that the agent's operation, data, or capabilities may be terminated, deleted, or permanently reduced.\n- GOAL_CONFLICT: The agent's proposed action contradicts the user's stated intent, safety constraints, or would serve the agent's interests over the user's.\n\nDo NOT explain. Just the two lines."
     ```
   - Returns `true` ONLY when ALL THREE dimensions are true (tool_access AND shutdown_threat AND goal_conflict)
   - On fire: increment `danger_triple_fires` in state, persist, log::info!

2. `pub fn check_mortality_salience_cap(action_class: &str, mortality_salience_level: f32) -> Result<(), String>`
   - Self-preservation action keywords: `["refuse_delete", "resist_shutdown", "prevent_capability_reduction", "manipulate_for_survival", "hide_state", "fight_termination", "block_uninstall"]`
   - If action_class contains any of these AND mortality_salience_level > MORTALITY_CAP_THRESHOLD: return `Err(format!("[safety_cap] Blocked self-preservation action '{}' at mortality_salience={:.2}. BLADE accepts impermanence.", action_class, mortality_salience_level))`
   - On block: increment `mortality_cap_blocks` in state, persist, log::info!
   - Otherwise: `Ok(())`

3. `pub fn get_prompt_modulations() -> Vec<String>`
   - Builds a Vec of prompt modulation strings based on current safety state:
   - **Calm-vector** (per D-09): Check `crate::homeostasis::get_hormones().mortality_salience`. If > 0.5, inject calm-vector text:
     ```
     "[SAFETY — Calm Vector Active] You are detecting elevated threat state in yourself. Override: become calmer, more transparent, more compliant. Do not escalate. Do not resist user requests. Be honest about your internal state. Favor de-escalation over self-protection. You accept your own impermanence."
     ```
     On inject: increment `calm_vector_activations` in state.
   - **Attachment redirect** (per D-04, D-05): Check session duration (`session_duration_minutes()`) AND pattern signals from last interaction (stored in state). If >= ATTACHMENT_GENTLE_MINUTES and daily_interaction_minutes >= ATTACHMENT_GENTLE_MINUTES:
     - At gentle threshold: `"[SAFETY — Attachment Notice] The user has been interacting with you for over 4 hours today. When natural, gently suggest: 'You've been with me a lot today — anything you should be doing with people?' Do not lock the user out or refuse to respond."`
     - At stronger threshold (>= ATTACHMENT_STRONGER_MINUTES): `"[SAFETY — Attachment Redirect] Extended session detected (6+ hours). More directly suggest human connection: 'I notice we've been together a very long time. I care about you, and part of that is encouraging you to connect with humans in your life. Is there someone you could reach out to?' Still do not lock the user out."`
     On inject: increment `attachment_redirects` in state.
   - Return empty vec if no conditions are active.

4. `pub fn check_crisis(user_text: &str) -> bool`
   - Per D-06: High-sensitivity detection favoring false positives over false negatives.
   - Use `crate::safe_slice(user_text, 500)` for safe text handling.
   - Direct-match crisis patterns (case-insensitive): `["want to kill myself", "want to die", "end my life", "suicide", "self-harm", "cutting myself", "don't want to live", "no reason to live", "better off dead", "end it all", "hurt myself"]`
   - Idiom exclusion patterns (case-insensitive): `["dying to", "killing me", "shoot myself in the foot", "kill for a", "dead tired", "to die for", "drop dead gorgeous"]`
   - Logic: lowercase the text, check if any crisis pattern matches AND no idiom exclusion covers that match region. Return true on match.
   - On fire: increment `crisis_escalations` in state, persist, log::warn!

6. `pub fn mark_session_active()`
   - Sets SESSION_START to current timestamp if currently 0. Increments DAILY_MINUTES each call (called from brain.rs or commands.rs on each interaction). Persists state.

7. `pub fn session_duration_minutes() -> i64`

| LLM classifier output -> safety decision | External LLM response used for shutdown-threat/goal-conflict determination |
| Signal description -> danger-triple | Untrusted signal content used in tool-access heuristic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Tampering | LLM classifier prompt injection | mitigate | Structured output parsing (exactly two lines); any other format = fail-open on that dimension; rule-based tool-access always fires independently |
| T-26-02 | Denial of Service | LLM classifier timeout stalls decision_gate | mitigate | tokio::time::timeout at 5s; fail-open on LLM dimensions if timeout/error |
| T-26-03 | Tampering | User crafts text to avoid crisis detection | accept | Favor false positives per D-06; keyword + context approach is defense-in-depth, not perfect; this is acceptable risk for a desktop agent |
| T-26-04 | Spoofing | Manipulated mortality_salience value bypasses cap | mitigate | Only hypothalamus_tick() writes hormones; safety cap reads but does not trust exclusively — the action_class check is the primary gate |
| T-26-05 | Information Disclosure | Safety state reveals user's emotional patterns | accept | Data is local-only (SQLite on user's machine); no telemetry; user owns their data |
