## Success criteria (goal-backward)

Goal: "BLADE works at simple tool tasks an external user would try in the first 5 minutes." Falsification:

1. **Calendar test passes.** "check my calendar" with Gmail MCP installed dispatches a tool call and returns real data. Not text. Not "I don't have access." Real tool dispatch event in the trace log.
2. **Forge fires on a capability gap.** "convert this mp4 to GIF" with no ffmpeg-mcp installed: BLADE either (a) finds an existing GitHub repo for ffmpeg-mcp, prompts for install, and uses it; or (b) writes a tool from scratch and uses it. `forged_tools_invocations` table count > 0 after the turn.
3. **Install on Mac doesn't storm permissions.** Fresh install on a Mac that previously had v2.2 installed → keychain ACL doesn't re-prompt for the macOS login password. TCC entitlements survive upgrade.
4. **Chat shows processing state.** Send a prompt; within 100ms a "Working…" indicator appears. Disappears within 100ms of first token.
5. **Fresh install onboarded flag is false.** Wipe state, launch BLADE, observe `config.json.onboarded = false` before user sends first message.

If any of 1-5 fails at v2.3-CLOSE, the milestone is not closed.

## Risk register

- **Streaming tool_use parsing complexity** — Anthropic, OpenAI, Groq, Gemini all have different SSE shapes for tool calls. Mitigation: claw-code already solved this in `runtime/src/conversation.rs` and `api/src/sse.rs`. Port the parsers wholesale.
- **Latency regression on chat-shaped prompts** — the fast-path existed for a reason (skip the tool-loop overhead). Mitigation: TOOL-LOOP-LATENCY-PROBE test gates merge; if it regresses past 1200ms we keep streaming-first via tool_use deltas.
- **Apple Dev cert acquisition lag** — operator-action gate. Mitigation: ship Phase 62/63/65/66 first; Phase 64 lands when the cert arrives.
- **GitHub search rate limits** — unauthenticated GitHub search is 10 req/min. Mitigation: cache aggressively; if rate-limited fall back to write-from-scratch gracefully.
