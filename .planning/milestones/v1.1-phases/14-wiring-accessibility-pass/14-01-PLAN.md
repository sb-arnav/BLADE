## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Rust backend → frontend event bus | blade_activity_log payload crosses IPC boundary; human_summary is user-controlled content |
| Frontend localStorage | Activity log entries written and read back; no sanitization needed (display-only, not executed) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01-01 | Tampering | ecosystem.rs emit_activity — human_summary | mitigate | `crate::safe_slice(summary, 200)` already applied in emit_activity; extend to all new emit call-sites added in this plan |
| T-14-01-02 | Info Disclosure | localStorage "blade_activity_log_v1" | accept | Local-only storage; no network sync; no PII in human_summary per 200-char cap and module naming convention |
| T-14-01-03 | Spoofing | ActivityLogProvider receives events from any Rust module | accept | Event bus is local IPC; no external origin can emit blade_activity_log into main window |
| T-14-01-04 | Denial of Service | Ring buffer overflow from high-frequency emits | mitigate | MAX_ENTRIES=500 cap enforced on every event; trim before write to localStorage |
</threat_model>
