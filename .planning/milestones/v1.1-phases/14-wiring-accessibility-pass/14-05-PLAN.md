  <how-to-verify>
Start the app with `npm run tauri dev` and perform these checks:

**1. Activity Log strip (LOG-01, LOG-03, LOG-04):**
  a. Navigate to Dashboard → Settings → Chat. The thin log strip (between TitleBar and content area) must remain visible on every route — it must NOT disappear or re-render on navigation.
  b. Trigger any BLADE action (e.g. send a chat message or toggle a Settings control). Within 2 seconds, the strip should update with a new entry showing module name and human_summary.
  c. Click the strip. The ActivityDrawer must open over the current route.
  d. Verify the drawer shows: module label, action verb, human_summary text, timestamp (HH:MM:SS format), and payload_id chip if non-null.
  e. Use the module filter dropdown to filter entries — verify the list updates.
  f. Close the drawer with the close button OR press Escape. Focus must return to the strip area.
  g. Restart BLADE (`npm run tauri dev` — full restart, not hot reload). Navigate back to Dashboard. The log strip should show the last N entries from before the restart (localStorage persistence). If the app was idle, this may be zero — trigger an action and restart again to confirm.

**2. Dashboard live data (WIRE2-02, WIRE2-03):**
  a. Navigate to Dashboard. Confirm NO card shows any of these texts:
     - "Tentacle reports + autonomy queue"
     - "Today's events + reminders"
     - "Connected services + status"
  b. Confirm all 3 card slots have headings: "Hive Signals", "Calendar", "Integrations".
  c. If Phase 12 scan has not run: the cards should show empty-state CTAs (e.g. "No tentacles active yet. Run deep scan to auto-configure.") — NOT "Coming Soon" or placeholder phase numbers.

**3. Settings pane additions (WIRE2-01):**
  a. Navigate to Settings → Voice. Confirm you can see: TTS Speed slider, Wake Word toggle, Whisper Model selector. Interact with the TTS Speed slider — verify it saves (no error toast).
  b. Navigate to Settings → Privacy. Confirm: Screen Timeline toggle, Capture Interval slider, Audio Capture toggle.
  c. Navigate to Settings → Appearance. Confirm: "God Mode" section with tier selector.

**4. Command palette reachability (WIRE2-04):**
  a. Open command palette (⌘K or Ctrl+K).
  b. Type "ghost". Confirm "Ghost Mode" appears as an entry.
  c. Type "lock". Confirm a lock screen entry is reachable.

**5. Keyboard navigation (A11Y2-01):**
  a. Navigate to Settings → Ecosystem using keyboard only (Tab through NavRail).
  b. Confirm all tentacle toggle checkboxes are reachable via Tab and activatable via Space.
  c. Navigate to Dashboard using keyboard only. Confirm the empty-state CTA buttons on TentacleSignalsCard and IntegrationsCard are focusable and activatable via Enter.
  d. Tab to the ActivityStrip. Confirm it is focusable (visible focus ring). Press Enter to open the drawer. Press Escape to close. Confirm focus returns to the strip.

**6. WCAG AA contrast (A11Y2-02):**
  a. With a light-colored wallpaper: check that the activity strip text is legible (dark text on semi-transparent background).
  b. Check the drawer heading and entry list — text must be clearly readable against the dialog background.
  c. Check the new Settings controls (tts_speed label, wake_word_enabled label) — label text must be readable.
  Note: Automated verify:contrast already ran in Task 1. This is a visual confirmation pass.
