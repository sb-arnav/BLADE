**FILE 1: `scripts/verify-spacing-ladder.mjs`**

Structure after verify-a11y-pass-2.mjs. Shebang `#!/usr/bin/env node`. ESM imports from `node:fs`, `node:path`, `node:url`.

Algorithm:

1. **Scope discovery** — walk these directories for `.css` files:
   - `src/features` (recursively)
   - `src/design-system` (recursively)
   - `src/styles` (recursively), EXCLUDING `tokens.css` and `layout.css` by name

2. **Whitelist (exact string match inside property value)** — build two whitelist Sets from SPACING-LADDER.md:
   - `VALUE_WHITELIST` = Set of literal value strings that are ALWAYS allowed regardless of selector:
     - `"0"`, `"1px"`, `"2px"`, `"3px"`
     - `"0 0"`, `"0 1px"`, `"0 2px"`, `"0 6px"`, `"0 10px"`, `"0 12px"`
   - `CHIP_WHITELIST_VALUES` = Set allowed only when the containing selector matches the chip regex:
     - `"3px 10px"`, `"4px 10px"`, `"4px 12px"`, `"6px 10px"`, `"6px 12px"`, `"8px 12px"`, `"8px 14px"`, `"8px 16px"`, `"10px 14px"`, `"12px 16px"`, `"10px 12px"`, `"2px 6px"`, `"1px 4px"`, `"1px 6px"`
   - `CHIP_SELECTOR_RE` = `/\.(chip|pill|badge|status|tlight|titlebar-traffic|tlight-|hormone-chip|dash-hero-state|dash-hero-chip|titlebar-status|titlebar-hint|coming-soon-card|voice-orb-window)/`

3. **File parsing** — for each CSS file:
   - Read content, split into lines
   - Track current selector block by matching lines that end with `{` and extracting the selector text (everything before `{` on that line OR preceding lines joined back to the last `}` or file start). A simple state: keep a `currentSelector` variable, updated when a line contains `{`.
   - For each line, regex test against the violation regex: `/^\s*(padding|padding-top|padding-bottom|padding-left|padding-right|margin|margin-top|margin-bottom|margin-left|margin-right|gap|row-gap|column-gap|grid-gap)\s*:\s*([^;]+);?/`
   - Extract property name and value string (trimmed)
   - Skip if value contains `var(--` (token use — always allowed)
   - Skip if value contains `calc(` with `var(--` inside (composite token use)
   - Skip if property is `margin` and value is `0` / `auto` / `0 auto` (exact string match)
   - Skip if value is in `VALUE_WHITELIST` (exact string match after trim)
   - If value is in `CHIP_WHITELIST_VALUES` AND `currentSelector` matches `CHIP_SELECTOR_RE`, skip
   - Otherwise: if value contains `px` or `rem` or a digit NOT preceded by `--`, flag as violation with `{ file, line, selector, property, value }`

4. **Report** — Print scan summary `[verify:spacing-ladder] Scanned N files across src/features, src/design-system, src/styles`. If zero violations, print `PASS` and exit 0. Otherwise print `FAIL — K violation(s):` and each as `  {file}:{line} — {selector} { {property}: {value}; } — not in spacing ladder or whitelist`.

5. **Exit contract:** 0 = pass, 1 = violations found.

**FILE 2: `scripts/verify-empty-states-copy.mjs`**

Shebang + ESM structure matching verify-a11y-pass-2.mjs.

Algorithm:

1. **Scope discovery** — walk `src/features/` for `.tsx` files.

2. **Banned bare-label patterns** (regex list — any match in the `label` prop triggers a check):
   ```js
   const BANNED_LABEL_RE = /(No\s+data|No\s+results|No\s+recent|Nothing\s+yet|Nothing\s+here)/i;
   const BARE_NEGATION_RE = /^\s*No\s+\w+(\s+yet)?\s*$/i;  // "No goals yet", "No items"
   ```

3. **Timeline/CTA allow-phrases** (regex — if any of these patterns appears on the same EmptyState, it passes):
   ```js
   const TIMELINE_RE = /(learning|give me|still|once|after|when|24h|48h|will appear|as BLADE|will populate|come back|start|add|connect|configure|enable)/i;
   ```

4. **Parsing strategy** — grep-style, multi-line:
   - Read each file, find every occurrence of `<EmptyState` and scan forward to the matching `/>` or `</EmptyState>` (crude: read until you see `/>` on its own line or `>` followed by `</EmptyState>`). A simpler approach: for each file, find every line that contains `<EmptyState` and collect lines until the opening tag closes (tracking `<` > `>` on subsequent lines).
   - Extract the JSX tag content as a string `tagContent`.
   - Check whether `tagContent` contains `actionLabel=` (PASS).
   - Check whether `tagContent` contains `description=` AND that description value matches `TIMELINE_RE` (PASS).
   - Extract `label="..."` or `label={"..."}` value. If the label value matches `TIMELINE_RE`, PASS.
   - Otherwise, test the label against `BANNED_LABEL_RE` or `BARE_NEGATION_RE`. If matches, flag as violation with `{ file, line, label }` — bare negation without any CTA/timeline/description escape hatch.

5. **Report** — `[verify:empty-states-copy] Scanned N TSX files`. On pass: `PASS — 0 bare-negation empty states`. On fail: `FAIL — K bare-negation empty state(s) missing CTA/timeline/description:` followed by per-violation lines.

6. **Exit contract:** 0 = pass, 1 = violations.
