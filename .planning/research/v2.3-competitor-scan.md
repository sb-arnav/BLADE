# v2.3 Competitor Scan — What to Lift

**Captured:** 2026-05-17
**Source:** web-researcher agent run, operator-authorized "look outside, mostly we can copy"
**Authority:** informs v2.3 REQUIREMENTS + v2.4+ VISION amendment proposal

## Sharpest single finding

**BLADE's "function call written as text" bug is a parser-commitment problem, not a routing problem.**

Hermes Function Calling repo makes this explicit: the model emits `<tool_call>` opener tokens or plain text depending on its decision. The inference layer must hold the parser in an "undecided" state until the discriminating token arrives, then switch handlers. The fast-path gate at `commands.rs:1822` commits to the plain-text parser before seeing the opener — that's the bug.

Fix is one condition, not a heuristic: **check for `<tool_call>` (Hermes) or `tool_use` block_start (Anthropic) as the first non-whitespace event in the assistant stream; if present, switch handlers.**

VLLM's Hermes parser already does this. Cline's `Task.ask()` suspension gate routes through it. Both Apache-2 / MIT.

## Patterns to lift, ranked by leverage

### 1. Hermes streaming gate (MIT — NousResearch/Hermes-Function-Calling)
- `<tool_call>{"name":"...","arguments":{...}}</tool_call>` is the canonical tool format
- `<tool_call>` is a special token in Hermes 3/4 — detectable mid-stream without buffering
- Parser combinator switching: undecided → text OR tool-accumulation on first discriminating token
- **Lift target:** `providers/anthropic.rs`, `providers/openai.rs` SSE parsers; new `streaming_gate.rs`

### 2. Goose AgentManager unification (Apache-2 — block/goose)
- One `AgentManager` keyed by `session_id` instead of branching code paths (interactive / scheduler / sub-recipe)
- "Everything is a Recipe at the boundary"
- BLADE already adopted Goose's Provider trait + session schema — AgentManager is the next layer
- **Lift target:** v2.4 candidate — collapse `send_message_stream_inline`'s branches into one `AgentSession` struct. Reference: Goose unification PRs #4389, #4684, #4828, #5082
- v2.3 is the surgical fix; v2.4 is the architectural collapse

### 3. OpenHands event-sourced conversation (MIT — arXiv:2511.03690)
- Append-only event log as the sole source of truth
- Tool calls = events; tool results = events; retries = new events, not mutations
- Replay, debugging, per-step approval become trivial
- BLADE's current mutable `conversation: Vec<ConversationMessage>` doesn't have this property
- **Lift target:** v2.4+ — architectural shift, not surgical fix

### 4. Cline `Task.ask()` suspension gate (Apache-2 — cline/cline, ~70k stars)
- Streaming handler routes each `tool_use` block through a suspension gate before dispatch
- Loop blocks until UI resolves the approval Promise
- Plan/Act mode split — now a user expectation, not a differentiator
- **Lift target:** v2.4 candidate when operator decides Cline-style per-step approval is on

### 5. Aider error-as-observation retry (Apache-2 — Aider-AI/aider)
- When LLM emits malformed output, send the parse error back as the next user turn
- No retry counter, no special retry state — loop continues with failure in context
- Beautiful primitive — "error becomes feedback"
- **Lift target:** v2.3 — wrap forge's "write tool from scratch" path; on syntax error, feed compiler stderr back as a user turn rather than a hardcoded retry

### 6. DeepHermes Atropos RL fine-tuning (Apache-2 — NousResearch/Atropos)
- 2.5-4.6x accuracy on parallel/simple tool calls over SFT base
- Reasoning-first + tool-calling as a two-system-prompt stack
- **Lift target:** v3.x — if BLADE ever ships a local-model default. Not v2.3.

## Cross-domain analogy worth noting

Parser combinator switching → BLADE fix. The Hermes streaming gate is structurally identical to how parser combinators handle ambiguous grammars: consume tokens eagerly until a discriminating token resolves which production rule applies, then switch parsers. The fast-path bug is premature parser commitment.

## What this changes in v2.3

The decisions.md 2026-05-17 entry says "rip the fast-path heuristic." The research sharpens that: **rip the heuristic AND add a streaming parser gate that switches on `<tool_call>` / `tool_use` block_start.** The two are complementary — the gate handles the streaming-first design without losing the latency advantage, the heuristic-rip ensures the dispatcher gets the chance to fire.

REQUIREMENTS.md Phase 62 is updated with this refinement.

## What this changes in v2.4+ VISION

Two amendments worth proposing:

1. **Event-sourced conversation as a primitive.** Currently VISION lists 5 primitives. OpenHands' event-sourcing pattern is a 6th candidate — "conversation is an immutable event log, replayable, per-step-approvable." Not a feature, an architectural primitive that makes a dozen downstream features cheap.

2. **Goose AgentManager + Recipe-as-session adoption.** v2.2 adopted Provider trait + session schema. v2.4 candidate: adopt AgentManager so BLADE's interactive chat, the scheduler (cron.rs morning briefings), and any future sub-recipes all share one execution path. Removes a class of "this code path silently broke" bugs.

Both deferred to v2.4 minimum — v2.3 must stay focused on the functionality fix.
